What You Will Do

This is the capstone lab. You will deploy your website to production, secure it, encrypt it with HTTPS, and set up automatic deployment from GitHub — all skills used by professional developers every day.

  • A live website at https://yourFQDN — no /iit path
  • SSL/HTTPS encryption via Let’s Encrypt
  • Selective password protection on sensitive labs (using your JSON secure flag)
  • Automatic deployment — push to GitHub, server updates itself

Using AI as a Tool

Throughout this lab, you’re encouraged to use AI coding tools (Claude Code, GitHub Copilot, ChatGPT) to generate code, debug issues, and learn concepts. This is how professional developers work — AI is a tool in your toolkit, just like VS Code or the terminal.

How to get good results from AI

TechniqueExample
Be specific about your environment “I’m on Ubuntu 22.04 with Apache2” — not just “my server”
Include error messages verbatim “I get this error: [paste exact text]” — not “it doesn’t work”
Give it your existing code “Here is my current JavaScript file. Modify it to add X.”
Ask it to explain, not just fix “Fix this and explain what was wrong so I understand.”
Verify the output Read the generated code. Does it make sense? Test it before deploying.

AI Prompts for Each Part

TaskPrompt
Generate the PHP redirect “Write a PHP file that does a 302 redirect to /iit/ using header().”
Add lock icons to projects page “Here is my JS that builds project cards from JSON. Add a lock icon when the secure field is true. [paste code]”
Write the deploy webhook “Write a PHP GitHub webhook handler that verifies X-Hub-Signature-256 with HMAC, only deploys pushes to main, runs git pull as www-data, and logs results.”
Debug Apache errors “Apache returns 500 after I moved .htaccess. Here’s my error log: [paste]. Fix it.”
Debug Certbot “Certbot --apache fails with [error]. My FQDN is X, Ubuntu 22.04, port 443 is open. What’s wrong?”
Azure CLI setup “Install Azure CLI on [macOS/Windows]. Show me commands to start/stop/status a VM in resource group student-dtlab.”
AI-generated code is a starting point, not a final answer. Always read what it produces, understand why it works, and test it before deploying to your server. If something doesn’t look right, ask the AI to explain it.

SSH • Azure VM Commands (after connecting)

# Git (always as www-data)
$ sudo -u www-data git pull origin main

# File operations
$ sudo mv source destination
$ sudo cp source destination
$ sudo chown -R www-data:www-data /var/www/html/iit

# SSL
$ sudo certbot --apache -d YOUR_FQDN
$ sudo certbot renew --dry-run

# Debugging
$ sudo tail -20 /var/log/apache2/error.log
$ sudo tail -20 /var/log/deploy.log
