Goal

Set up Continuous Deployment so that when you push to GitHub, your server automatically pulls the latest code. No more SSH → git pull every time you make a change.

The flow:

  1. You push to GitHub (as normal via GitHub Desktop)
  2. GitHub POSTs a webhook to a URL on your server
  3. Your server verifies the request with a shared secret (HMAC-SHA256), then runs git pull

This is the same concept behind GitHub Actions, Jenkins, and other CI/CD systems used in industry.

Create the Webhook Handler

A PHP script that receives GitHub pushes and deploys

Step 4.1 — Generate a secret

SSH • Azure VM
$ openssl rand -hex 16

Copy the output — this is your webhook secret. You’ll use it in both the PHP script and on GitHub.

Save this secret somewhere safe. You’ll need it for Step 4.3 (GitHub webhook configuration).

Step 4.2 — Create deploy.php

SSH • Azure VM
Try with AI

Have the AI write this script for you. Give it the exact requirements:

“Write a PHP webhook handler called deploy.php for GitHub. It should: (1) verify the X-Hub-Signature-256 header using a shared secret with hash_hmac, (2) only deploy on pushes to the main branch by checking the ref field in the JSON payload, (3) run git pull origin main as www-data in /var/www/html/iit, (4) log every attempt (success or failure) to /var/log/deploy.log with a timestamp. Return 403 for bad signatures and 200/500 for success/failure.”

Review the code the AI gives you. Verify it checks the signature before doing anything else. Compare it to the reference implementation below.

$ sudo nano /var/www/html/deploy.php

Reference implementation (your AI-generated version should look similar):

<?php
// GitHub Webhook Auto-Deploy

$secret   = 'YOUR_SECRET_HERE';  // Replace with your secret from Step 4.1
$repoPath = '/var/www/html/iit';
$logFile  = '/var/log/deploy.log';

// Verify GitHub's HMAC-SHA256 signature
$signature = $_SERVER['HTTP_X_HUB_SIGNATURE_256'] ?? '';
$payload   = file_get_contents('php://input');
$expected  = 'sha256=' . hash_hmac('sha256', $payload, $secret);

if (!hash_equals($expected, $signature)) {
    http_response_code(403);
    file_put_contents($logFile, date('Y-m-d H:i:s') . " REJECTED: invalid signature\n", FILE_APPEND);
    die('Invalid signature');
}

// Only deploy pushes to main
$data   = json_decode($payload, true);
$branch = $data['ref'] ?? '';

if ($branch !== 'refs/heads/main') {
    file_put_contents($logFile, date('Y-m-d H:i:s') . " SKIPPED: push to $branch\n", FILE_APPEND);
    die('Not main branch');
}

// Deploy
$output = [];
$code   = 0;
exec("cd $repoPath && sudo -u www-data git pull origin main 2>&1", $output, $code);

$result = implode("\n", $output);
$status = $code === 0 ? 'SUCCESS' : 'FAILED';
file_put_contents($logFile, date('Y-m-d H:i:s') . " $status: $result\n", FILE_APPEND);

http_response_code($code === 0 ? 200 : 500);
echo $status;
?>

Replace YOUR_SECRET_HERE with the secret you generated in Step 4.1.

Step 4.3 — Set file permissions and create the log

SSH • Azure VM
$ sudo chown www-data:www-data /var/www/html/deploy.php
$ sudo chmod 640 /var/www/html/deploy.php
$ sudo touch /var/log/deploy.log
$ sudo chown www-data:www-data /var/log/deploy.log

Configure Server Permissions

Allow Apache to run git pull — and nothing else

Step 4.4 — Configure sudoers for www-data

SSH • Azure VM

Apache (running as www-data) needs permission to run git pull without a password prompt. We grant only that command — nothing else. This is the principle of least privilege.

$ sudo visudo -f /etc/sudoers.d/www-data-git

Add this single line:

www-data ALL=(www-data) NOPASSWD: /usr/bin/git pull*

Save: Ctrl+O, Enter, Ctrl+X.

Connect GitHub to Your Server

Add the webhook and test the full pipeline

Step 4.5 — Add the webhook on GitHub

Browser
  1. Your repo on GitHub.com → Settings → Webhooks → Add webhook
  2. Payload URL: https://yourFQDN/deploy.php
  3. Content type: application/json
  4. Secret: paste the secret from Step 4.1
  5. SSL verification: Enable (this works because you set up SSL in Part 3)
  6. Events: “Just the push event”
  7. Click Add webhook
