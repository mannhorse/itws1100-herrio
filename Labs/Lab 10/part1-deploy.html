Prepare Your Code Locally

Branch, create the redirect, update listings, merge

Step 1.1 — Create a lab10 branch

Choose your method
  1. Make sure your ITWS1100-yourRCSid repo is selected and you’re on main
  2. Click Fetch origin
  3. Branch → New Branch → name it lab10 → Create Branch
CLI Local
$ git checkout main && git pull origin main && git checkout -b lab10

Branch dropdown (or git branch) shows lab10.

Step 1.2 — Create the lab10 folder and redirect file

Local

Open your repo folder (Repository → Show in Finder/Explorer in GitHub Desktop, or just navigate to it). Create a lab10 folder. Inside it, create index.php with:

<?php
// header() sends a server-level 302 redirect before any HTML loads
// This MUST be the very first thing in the file - no whitespace above
header("Location: /iit/");
exit;
?>

Compare what the AI gives you to the code above. Does it match? Did it add anything extra?

Deploy to Your Server

Pull code, set up landing page, clean up

Step 1.7 — Start your VM

Choose your method

Navigate to your VM on portal.azure.com and click Start if it’s stopped.

CLI Local
$ az login
$ az vm start --resource-group student-dtlab --name yourRCSidRPI
$ az vm show -d --resource-group student-dtlab --name yourRCSidRPI --query "powerState" -o tsv
Try with AI
“How do I install the Azure CLI on [macOS/Windows]? I want to start, stop, and check the status of a VM named [yourRCSidRPI] in resource group student-dtlab from the command line.”

Step 1.8 — SSH in and pull your code

Local

Open a terminal (GitHub Desktop: Repository → Open in Terminal). Connect to your server:

$ ssh yourRCSid@yourRCSidRPI.eastus.cloudapp.azure.com
SSH • Azure VM

Once connected, pull your code:

$ cd /var/www/html/iit
$ sudo -u www-data git pull origin main
Remember: always sudo -u www-data for git on the server. See Lab 1, Tutorial 4 (GitHub and Azure) for why.
Run ls lab10/ — you should see index.php and README.md.

Step 1.9 — Set up the landing page redirect

SSH • Azure VM

Replace the default Apache page with your redirect so yourFQDN goes to your homepage.

$ cd /var/www/html
$ sudo mv index.html index.html.shipped      # Backup the default page
$ sudo cp iit/lab10/index.php ./index.php     # Your redirect takes its place
$ sudo chown www-data:www-data index.php
Browser In a private window, go to http://yourFQDN (no /iit). Should redirect to your homepage.

Step 1.10 — Clean up your repo

Local

Open your repo folder. Move any inclass, quiz, and .zip files out of your repo to a backup folder on your Desktop. These don’t belong on a production site.

Step 1.11 — Commit the cleanup and deploy

Choose your method to commit & push
  1. Review deleted files in Changes
  2. Summary: lab10: clean up inclass, quiz, and zip files
  3. Commit to main → Push origin
CLI Local
$ git add -A && git commit -m "lab10: clean up inclass, quiz, and zip files"
$ git push origin main

Then pull on the server:

SSH • Azure VM
$ cd /var/www/html/iit
$ sudo -u www-data git pull origin main
Browser Full site test (private window):
  • http://yourFQDN → redirects to homepage
  • Projects page shows all labs
  • Lab 7 links to group project
