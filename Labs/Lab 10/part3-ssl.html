Lab 10 · Part 3 — SSL / HTTPS

ITWS 1100 — Lab 10 Overview

Goal

Encrypt your site with HTTPS using a free Let’s Encrypt certificate. After this part, your site will show a padlock in the browser and all traffic — including the .htaccess passwords from Part 2 — will be encrypted.

Without SSL, everything travels in plain text. Anyone on the same network could intercept the passwords your visitors type into the .htaccess prompts. SSL encrypts the entire connection between the browser and your server.

We’ll use Certbot to get a free certificate from Let’s Encrypt and auto-configure Apache.

Step 3.1 — Open port 443 on Azure

Choose your method

Your VM → Networking → Network Settings → Create port rule → Inbound port rule:

  • Destination port: 443
  • Protocol: TCP
  • Name: Allow-HTTPS
CLI Local
# Find your NSG name, then add the rule
$ az network nsg list --resource-group student-dtlab --query "[].name" -o tsv
$ az network nsg rule create \
    --resource-group student-dtlab \
    --nsg-name NSG_NAME \
    --name Allow-HTTPS \
    --priority 1010 \
    --destination-port-ranges 443 \
    --protocol TCP --access Allow
Try with AI
“I need to open port 443 (HTTPS) on my Azure VM. The VM is in resource group student-dtlab. Show me how to do it in [the Azure Portal / Azure CLI]. Include the NSG inbound rule settings.”

Step 3.2 — Install Certbot and get your certificate

SSH • Azure VM
$ sudo apt update
$ sudo apt install -y certbot python3-certbot-apache
$ sudo certbot --apache -d YOUR_FQDN

Certbot will ask you a few questions:

  • Email: your RPI email (for renewal notices)
  • Terms of service: agree
  • Share email with EFF: your choice

Certbot automatically configures Apache and sets up HTTP → HTTPS redirect.

Try with AI

If Certbot fails with “Could not automatically find a matching server name”:

“Certbot can't find my ServerName. I'm on Ubuntu 22.04 with the default Apache config. My FQDN is [yourFQDN]. Write me the exact commands to add ServerName to my Apache config and restart Apache so Certbot will work.”
Browser
  • https://yourFQDN — shows a padlock in the address bar
  • http://yourFQDN — automatically redirects to https://

Step 3.3 — Verify auto-renewal

SSH • Azure VM

Let’s Encrypt certificates expire every 90 days. Certbot sets up a systemd timer to automatically renew them. Verify it’s working:

$ sudo certbot renew --dry-run
$ sudo systemctl status certbot.timer
Dry run says “all simulated renewals succeeded.” Timer shows active (waiting).
Your VM must be running for auto-renewal to work. If your VM is stopped when the certificate expires (90 days), you’ll need to start it and run sudo certbot renew manually.
