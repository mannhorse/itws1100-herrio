Goal

Reorganize your .htaccess security from protecting the entire site to protecting only specific folders. Add a secure flag to your JSON and show lock icons on your projects page so visitors know which labs require a password.

Reorganize .htaccess Security

Move from blanket protection to per-folder security

Your .htaccess in iit/ currently protects the entire site. We need it only on folders with sensitive content. An .htaccess file protects the folder it’s in and everything below it — this is called cascading.

See Lab 1, Tutorial 3 for how .htaccess and .htpasswd work together.

Step 2.1 — Move .htaccess out of the root

SSH • Azure VM
$ cd /var/www/html/iit
$ sudo mv .htaccess lab10/.htaccess.original
Browser http://yourFQDN/iit/ should load without a password prompt now.

Step 2.2 — Protect specific folders

SSH • Azure VM

Copy .htaccess into each folder that should require a password. At minimum, protect lab01 and lab09:

$ sudo cp lab10/.htaccess.original lab01/.htaccess
$ sudo cp lab10/.htaccess.original lab09/.htaccess
# Add more folders as needed
$ sudo chown -R www-data:www-data /var/www/html/iit
Browser (private window):
  • /iit/ — no password
  • /iit/lab02/ — no password
  • /iit/lab01/ — asks for password
  • /iit/lab09/ — asks for password

JSON Secure Flag & Lock Icons

Mark secured labs in your data and show it on the projects page

Now that certain folders are password-protected on the server, you need to tell visitors which ones require a password. You’ll do this by adding a secure field to your JSON data and updating your JavaScript to show a lock icon.

Step 2.3 — Add the secure field to your JSON

Local

Open your JSON file from Lab 8 (e.g., menuItems.json or whatever you named yours). Add a "secure" field to every entry — not just the new ones.

File: menuItems.json (your filename may differ)

{
  "menuItem": [
    {
      "menuName": "Lab 1",
      "menuDesc": "Azure & GitHub Setup",
      "menuURL": "lab01/",
      "secure": true
    },
    {
      "menuName": "Lab 2",
      "menuDesc": "HTML Resume",
      "menuURL": "lab02/",
      "secure": false
    },
    ... all your other labs ...
    {
      "menuName": "Lab 9",
      "menuDesc": "PHP & MySQL",
      "menuURL": "lab09/",
      "secure": true
    },
    {
      "menuName": "Lab 10",
      "menuDesc": "Production Deployment",
      "menuURL": "lab10/",
      "secure": false
    }
  ]
}
Your field names may be different (a, b, c or menuName, menuDesc, menuURL) — use whatever names you already have. Just add "secure" to each entry.
The folders you protect with .htaccess (Step 2.2) must match the entries you mark "secure": true here. They work together: the JSON flag shows the lock icon on your projects page, and the .htaccess file enforces the actual password on the server.

Step 2.4 — Show lock icons on the projects page

Local

Open the external JavaScript file that reads your JSON and builds the projects page (from Lab 8 — e.g., lab8.js). You need to modify the part where it builds each list item to check the secure field and add a lock icon.

File: lab8.js (your filename may differ)

Find the line where you build the <li> for each menu item. It probably looks something like this:

// Your current code (approximately)
$.each(responseData.menuItem, function(i, item) {
  menuli += '<li><a href="' + item.menuURL + '">';
  menuli += item.menuName + '</a> ';
  menuli += '<span class="description">' + item.menuDesc + '</span>';
  menuli += '</li>';
});

Add a check for item.secure and insert a lock icon when it’s true:

// Updated code with lock icon
$.each(responseData.menuItem, function(i, item) {
  var lockIcon = item.secure ? ' 🔒' : '';
  menuli += '<li><a href="' + item.menuURL + '"';
  if (item.secure) {
    menuli += ' title="Password required"';
  }
  menuli += '>';
  menuli += item.menuName + lockIcon + '</a> ';
  menuli += '<span class="description">' + item.menuDesc + '</span>';
  menuli += '</li>';
});

This adds a lock emoji (🔒) next to the project name and a “Password required” tooltip when hovering over secured links.
